🔧 Installation de XAMPP sur Ubuntu

1 Téléchargement de XAMPP
# Télécharger XAMPP pour Linux wget https://sourceforge.net/projects/xampp/files/XAMPP%20Linux/8.2.12/xampp-linux-x64-8.2.12-0-installer.run # Rendre le fichier exécutable chmod +x xampp-linux-x64-8.2.12-0-installer.run
2 Installation
# Lancer l'installation avec les droits administrateur sudo ./xampp-linux-x64-8.2.12-0-installer.run
ℹ️ Suivez les instructions de l'installateur graphique. L'installation par défaut se fait dans /opt/lampp
3 Démarrage de XAMPP
# Démarrer tous les services sudo /opt/lampp/lampp start # Vérifier le statut sudo /opt/lampp/lampp status
✅ Vous devriez voir : "Apache is running" et "MySQL is running"

Commandes Utiles XAMPP

# Arrêter XAMPP sudo /opt/lampp/lampp stop # Redémarrer XAMPP sudo /opt/lampp/lampp restart # Démarrer uniquement Apache sudo /opt/lampp/lampp startapache # Démarrer uniquement MySQL sudo /opt/lampp/lampp startmysql

🗄️ Configuration de la Base de Données

1 Accès à MySQL
# Se connecter à MySQL /opt/lampp/bin/mysql -u root -p # Ou via phpMyAdmin http://localhost/phpmyadmin
2 Création de la base de données
-- Créer la base de données CREATE DATABASE gestioncharge_mosca CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci; -- Utiliser la base USE gestioncharge_mosca;
3 Création de la table Utilisateurs
CREATE TABLE Utilisateurs ( id INT AUTO_INCREMENT PRIMARY KEY, nom VARCHAR(100) NOT NULL, prenom VARCHAR(100) NOT NULL, email VARCHAR(150) NOT NULL UNIQUE, mot_de_passe VARCHAR(255) NOT NULL, role ENUM('admin', 'technicien', 'utilisateur') DEFAULT 'utilisateur', date_creation TIMESTAMP DEFAULT CURRENT_TIMESTAMP, derniere_connexion TIMESTAMP NULL, actif BOOLEAN DEFAULT TRUE, INDEX idx_email (email), INDEX idx_role (role) ) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;
4 Création de la table Donnees
CREATE TABLE `Donnees` ( `Processus` varchar(40) NOT NULL, `Tache` varchar(200) NOT NULL, `Charge` decimal(15,2) DEFAULT NULL, `Date` date DEFAULT NULL ) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_general_ci;

🔧 Dépannage

Problème : MySQL ne démarre pas

# Vérifier les logs sudo tail -50 /opt/lampp/var/mysql/*.err # Réparer les permissions sudo chown -R mysql:mysql /opt/lampp/var/mysql # Redémarrer sudo /opt/lampp/lampp restart

Problème : Erreur de connexion socket

Utilisez 127.0.0.1 au lieu de localhost dans config.php

Problème : Page 404

  • Vérifier que XAMPP est démarré : sudo /opt/lampp/lampp status
  • Vérifier les permissions du dossier gestioncharge
  • Vérifier que le fichier index.php existe

🔒 Recommandations de Sécurité

Pour la mise en production :

  • Définir un mot de passe pour l'utilisateur root MySQL
  • Créer un utilisateur MySQL dédié avec droits limités
  • Utiliser HTTPS (SSL/TLS)
  • Implémenter l'authentification des utilisateurs
  • Valider toutes les entrées utilisateur
  • Utiliser des requêtes préparées (PDO)
  • Sauvegarder régulièrement la base de données

Créer un utilisateur MySQL dédié

-- Créer un utilisateur spécifique CREATE USER 'mosca_user'@'localhost' IDENTIFIED BY 'MotDePasseSecurise123!'; -- Donner les droits sur la base gestioncharge_mosca uniquement GRANT SELECT, INSERT, UPDATE, DELETE ON gestioncharge_mosca.* TO 'mosca_user'@'localhost'; -- Appliquer les changements FLUSH PRIVILEGES;

💾 Sauvegarde de la Base de Données

Créer une sauvegarde complète

# Sauvegarde de toute la base /opt/lampp/bin/mysqldump -u root gestioncharge_mosca > backup_gestioncharge_$(date +%Y%m%d).sql # Sauvegarde compressée /opt/lampp/bin/mysqldump -u root gestioncharge_mosca | gzip > backup_gestioncharge_$(date +%Y%m%d).sql.gz

Restaurer une sauvegarde

# Restaurer depuis un fichier SQL /opt/lampp/bin/mysql -u root gestioncharge_mosca < backup_gestioncharge_20241228.sql
